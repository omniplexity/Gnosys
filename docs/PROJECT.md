# Gnosys Project

## Summary

Gnosys is a desktop-native, chat-first agent workspace with a working backend runtime, local persistence, layered memory, orchestration, policy controls, and operational UI surfaces.

## Current state

- desktop shell is implemented
- backend runtime is implemented
- persistence and event logging are implemented
- memory retrieval and consolidation are implemented
- orchestration and worker spawning are implemented
- CRUD for tasks, projects, agents, skills, and schedules is implemented
- autonomy and approval controls are implemented
- diagnostics replay and memory review are implemented

## Stage

The repository is in the operational core stage.

- the foundation is complete
- the next work focuses on automation maturity, governance depth, and evaluation

## What remains

- schedule daemon and background recurrence
- stronger memory promotion and contradiction handling
- skill lifecycle workflows beyond CRUD
- richer replay, diffing, and metrics
- formal evaluation and regression harnesses
